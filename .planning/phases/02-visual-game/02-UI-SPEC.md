## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Card corner text padding, inline icon gaps |
| sm | 8px | Gap between cards in a hand, button internal padding (vertical) |
| md | 16px | Default element spacing, button internal padding (horizontal) |
| lg | 24px | Zone internal padding (dealer zone, player zone, controls zone) |
| xl | 32px | Gap between major table zones |
| 2xl | 48px | Vertical separation between dealer zone and player zone |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Card gap within a hand: 12px horizontal overlap offset (cards fan with partial overlap, not full 8px separation)
- Touch target minimum: 44px height for all interactive buttons (chip buttons, action buttons, deal button)

### Animation States

| Game Phase | Animations Active | Buttons State |
|------------|-------------------|---------------|
| BETTING | None | Chip buttons + Deal enabled; action buttons disabled |
| DEALING | Card deal slide-in | ALL buttons disabled |
| PLAYER_TURN | None (between actions) | Action buttons per `getAvailableActions()`; chip buttons disabled |

### Error States

This phase has no network calls or async data loading. Error conditions are handled by game logic:

| Condition | Handling |
|-----------|----------|
| Bet exceeds chip balance | Chip button disabled when adding would exceed balance |
| Bet below minimum ($10) | Deal button disabled until bet >= $10 |
| Bet above maximum ($500) | Chip buttons disabled when bet reaches $500 |
| No chips remaining | Bankrupt state: "Out of chips!" + "New Session" button |

No destructive user actions exist in this phase. "New Session" resets chips but there is no confirmation needed (per BET-05, player can start new session at any time).

## Interaction Contract

### Betting Phase
1. Player clicks chip buttons to accumulate bet amount
2. Each click adds denomination to current bet (no debounce)
3. Chip buttons disable individually when adding that denomination would exceed balance or max bet
4. "Clear" resets bet to $0
5. "Deal" is enabled only when bet >= $10; clicking triggers `engine.placeBet()` then `engine.deal()`
6. Transition: BETTING -> DEALING (deal animation plays)

### Player Turn
1. Action buttons reflect `engine.getAvailableActions()` after each action
2. "Hit" calls `engine.hit()`, new card animates in from shoe
3. "Stand" calls `engine.stand()`, transitions to dealer turn
4. "Double" calls `engine.doubleDown()`, one card animates in, then dealer turn
5. If player busts, skip to ROUND_OVER immediately (no dealer turn per Phase 1 decision)

### Dealer Turn
1. Hole card flips (300ms 3D rotation)
2. Dealer draws cards per engine logic, each animates in from shoe
3. All buttons disabled throughout

### Round Over
1. Result banner appears with outcome text
2. Banner auto-dismisses after 2000ms
3. Discard sweep animation runs (all cards slide to discard pile)
4. UI returns to BETTING phase, bet reset to $0, ready for next hand
