Create `src/ui/AnimationManager.js`:

```javascript
import { ANIM } from '../constants.js';

export class AnimationManager {
  #busy = false;

  get isBusy() { return this.#busy; }
```

**Methods:**

1. `async dealCards(cardElements, shoeEl, targetPositions)` — Animate cards from shoe to hand positions.
   - Set `this.#busy = true`
   - Get shoe position via `shoeEl.getBoundingClientRect()`
   - For each card element (in deal order: player card 1, dealer card 1, player card 2, dealer card 2):
     - Position card at shoe coordinates using `style.position = 'fixed'`, `style.left`, `style.top` (shoe's bounding rect)
     - Set `style.opacity = '1'` to make visible
     - Force reflow: read `element.offsetHeight`
     - Set `style.transition = 'transform ${ANIM.DEAL_DURATION}ms ease-out, left ${ANIM.DEAL_DURATION}ms ease-out, top ${ANIM.DEAL_DURATION}ms ease-out'`
     - Set `style.left` and `style.top` to `targetPositions[i]` (the card's final position in the hand area)
     - Wait for transition via `this.#waitForTransition(element, ANIM.DEAL_DURATION)`
     - After transition, switch card to `position: absolute` within its hand container (remove fixed positioning, set `style.left` to the overlap offset within the container)
     - Wait `ANIM.DEAL_STAGGER` ms before next card (except after last)
   - Set `this.#busy = false`

   **Simpler approach (recommended):** Instead of fixed positioning math, use CSS transforms:
   - Each card element is already inside its hand container (placed by TableRenderer)
   - Start each card with `style.opacity = '0'` and `style.transform = 'translate(Xpx, Ypx)'` where X,Y is the offset from card's position to the shoe
   - Calculate offset: `shoeRect.left - cardRect.left`, `shoeRect.top - cardRect.top`
   - Force reflow
   - Set transition and transform to `translate(0, 0)` + opacity to `1`
   - Wait for transitionend

2. `async flipCard(cardElement)` — 3D flip for dealer hole card reveal.
   - Set `this.#busy = true`
   - Remove class `bj-card--flipped` from the card element (CSS transition on `.bj-card-inner` handles the 3D rotation via `--bj-flip-duration`)
   - Wait for transition via `this.#waitForTransition(cardElement.querySelector('.bj-card-inner'), ANIM.FLIP_DURATION)`
   - Set `this.#busy = false`

3. `async discardAll(cardElements, discardEl)` — Sweep cards to discard pile.
   - Set `this.#busy = true`
   - Get discard pile position via `discardEl.getBoundingClientRect()`
   - For each card element:
     - Get card's current position via `getBoundingClientRect()`
     - Calculate offset to discard pile
     - Set `style.transition = 'transform ${ANIM.DISCARD_DURATION}ms ease-in, opacity ${ANIM.DISCARD_DURATION}ms ease-in'`
     - Set `style.transform = 'translate(Xpx, Ypx)'` toward discard pile
     - Set `style.opacity = '0.5'`
     - Do NOT await — let cards overlap for visual effect
     - Wait `ANIM.DISCARD_STAGGER` ms before starting next card
   - After last card starts, wait `ANIM.DISCARD_DURATION` ms for it to complete
   - Set `this.#busy = false`

4. `async slideCardIn(cardElement, shoeEl)` — Single card animation for hit/double (similar to deal but one card).
   - Set `this.#busy = true`
   - Same transform approach as dealCards but for a single card
   - Set `this.#busy = false`

5. `#waitForTransition(element, duration)` — Private helper. Returns a Promise.
   - Listen for `transitionend` event with `{ once: true }`
   - Also set a safety timeout at `duration + 50` ms that resolves if transitionend never fires
   - Whichever fires first resolves the promise (clear the other)

6. `#delay(ms)` — Private helper. Returns `new Promise(resolve => setTimeout(resolve, ms))`.

**Update `src/ui/UIController.js`:**

1. Add import: `import { AnimationManager } from './AnimationManager.js';`

2. In constructor, create `this.animManager = new AnimationManager();`

3. Replace `this.animBusy = false` usage — now use `this.animManager.isBusy` for the animBusy flag passed to renderControls.

4. **Make the Deal handler async:**
   ```
   async #handleDeal():
     - Call engine.placeBet(pendingBet), engine.deal()
     - Reset pendingBet = 0
     - Render state (cards now exist in DOM but invisible — opacity 0 from cards.css)
     - Get card elements from DOM: collect all .bj-card elements from dealerHandEl and playerHand0El
     - Build deal order array: [player-card-0, dealer-card-0, player-card-1, dealer-card-1]
     - Get shoe element reference
     - Compute target positions for each card (their position in the hand container)
     - Render controls with animBusy = true (disable all buttons)
     - await animManager.dealCards(cardElements, shoeEl, targetPositions)
     - Re-render controls with animBusy = false
     - Check if state is ROUND_OVER (blackjack): if so, handle round over
   ```

5. **Make the Hit handler async:**
   ```
   async #handleHit():
     - Render controls with animBusy = true
     - Call engine.hit()
     - Render new state (new card in DOM but invisible)
     - Get the newly added card element (last .bj-card in playerHand0El)
     - await animManager.slideCardIn(newCardEl, shoeEl)
     - Re-render controls
     - If state is ROUND_OVER (bust), handle round over
   ```

6. **Make the Stand handler async:**
   ```
   async #handleStand():
     - Render controls with animBusy = true
     - Call engine.stand() — this processes dealer turn in engine (all at once)
     - Now animate the reveal:
       - Get the dealer's hole card element (second .bj-card in dealerHandEl)
       - await animManager.flipCard(holeCardEl)
       - Update dealer hand rendering to show revealed cards
       - If dealer drew additional cards (dealerHand.cards.length > 2):
         - For each additional card beyond the initial 2, render and animate slide-in
     - Re-render with final state
     - Handle round over
   ```

   **Important design note:** The engine processes the entire dealer turn synchronously (stand() returns final state). The UI must animate the reveal step by step. Approach:
   - Before calling stand(), save `dealerCardCount = state.dealerHand.cards.length` (should be 2)
   - Call engine.stand() — get final state with all dealer cards
   - Animate hole card flip first
   - Then for each dealer card beyond the initial 2, create card element, add to dealer hand container, animate slide-in
   - After all dealer cards animated, show final scores

7. **Make the Double handler async:**
   ```
   async #handleDouble():
     - Render controls with animBusy = true
     - Call engine.doubleDown()
     - Render state
     - Get new card element, animate slide-in
     - If not bust, animate dealer turn (same as stand flow)
     - Handle round over
   ```

8. **Update #handleRoundOver to include discard animation:**
   ```
   async #handleRoundOver():
     - Show result banner (await renderer.showResult)
     - Collect all .bj-card elements from dealer and player hand areas
     - await animManager.discardAll(cardElements, discardEl)
     - Call renderer.clearTable()
     - If chips > 0: engine.startNewRound(), render
     - Else: render bankrupt state
   ```

9. **Update all button event listeners** to check `animManager.isBusy` before acting. If busy, return immediately (do nothing on click).
