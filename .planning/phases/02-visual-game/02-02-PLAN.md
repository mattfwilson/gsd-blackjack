**Structure:**

The UIController class manages:
- `engine` — GameEngine instance
- `sound` — SoundManager instance
- `renderer` — TableRenderer instance
- `pendingBet` — accumulated bet in cents before Deal is pressed (starts at 0)
- `animBusy` — boolean flag, false for now (Plan 03 adds AnimationManager)

**Constructor:**
- Create engine, sound, renderer instances
- Call `syncAnimToCSS()`
- Call `this.#bindEvents()`
- Call `this.#render()` for initial state

**Private method `#bindEvents()`:**
- Chip buttons (`.bj-chip-btn`): on click, read `data-amount` (parseInt), add to `pendingBet` if it wouldn't exceed balance (engine.getState().chips + pendingBet is the available pool — but engine already deducted chips on placeBet, so during BETTING chips = full balance). Validate: `pendingBet + amount <= engine.getState().chips` and `pendingBet + amount <= 50000`. If valid, increment `pendingBet`, re-render.
- Clear button (`.bj-btn-clear`): reset `pendingBet = 0`, re-render.
- Deal button (`.bj-btn-deal`): if `pendingBet >= 1000`, call `engine.placeBet(pendingBet)` then `engine.deal()`, reset `pendingBet = 0`, render. If state is ROUND_OVER immediately (blackjack scenarios), call `this.#handleRoundOver()`.
- Hit button (`.bj-btn-hit`): if not animBusy, call `engine.hit()`, render. If state becomes ROUND_OVER (bust), call `this.#handleRoundOver()`.
- Stand button (`.bj-btn-stand`): if not animBusy, call `engine.stand()`, render. Result goes through dealer turn and resolves — state will be ROUND_OVER, call `this.#handleRoundOver()`.
- Double button (`.bj-btn-double`): if not animBusy, call `engine.doubleDown()`, render. State may be ROUND_OVER, call `this.#handleRoundOver()`.

**Also handle bankrupt state:** When controls render detects chips === 0 and phase === ROUND_OVER, the "New Session" button should exist. Bind a delegated click listener on `.bj-zone-controls` that checks if the clicked element is the New Session button (or has a data attribute), then calls `engine.resetSession()`, `pendingBet = 0`, re-render.

**Private method `#handleRoundOver()`:**
- Get state, get result
- Determine if player bust: state.playerHands[0].isBust
- Call `renderer.showResult(result, currentBet, isBust)` which returns a promise
- After promise resolves (banner auto-clears), call `renderer.clearTable()`
- If chips > 0: call `engine.startNewRound()`, re-render (back to BETTING)
- If chips === 0: re-render (shows bankrupt state)

**Private method `#render()`:**
- Get `state = engine.getState()`
- Get `actions = engine.getAvailableActions()`
- Call `renderer.renderChips(state.chips)`
- Call `renderer.renderBet(state.phase === 'BETTING' ? pendingBet : state.currentBet)` — during BETTING show accumulated pendingBet, during play show locked-in currentBet
- If state has playerHands[0]: `renderer.renderHand(renderer.playerHand0El, state.playerHands[0].cards)` and render player score
- If state has dealerHand with cards: `renderer.renderHand(renderer.dealerHandEl, state.dealerHand.cards)` and render dealer score
- Call `renderer.renderControls(state.phase, actions, this.animBusy, this.pendingBet, state.chips)`

**Boot at module level:**
```javascript
const controller = new UIController();
```
