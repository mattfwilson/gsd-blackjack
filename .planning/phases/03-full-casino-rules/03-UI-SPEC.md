# Phase 3 — UI Design Contract

> Visual and interaction contract for the full casino rules phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla JS/CSS/HTML) |
| Icon library | none (Unicode suit symbols) |
| Font | system-ui, -apple-system, "Segoe UI", sans-serif |

**Note:** This is a vanilla JS/CSS/HTML project with zero external dependencies. All styling uses `bj-` prefixed CSS classes established in Phase 2. Phase 3 extends the existing design system -- it does not introduce a new one.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, card corner padding |
| sm | 8px | Compact element spacing, chip-row gap, action-row gap, hand-group gap |
| md | 16px | Default element spacing, shoe-area gaps |
| lg | 24px | Section padding (zone padding, status bar horizontal padding) |
| xl | 32px | Result banner horizontal padding |
| 2xl | 48px | Player zone hand-area gap (between hand 0 and hand 1) |
| 3xl | 64px | Not used in Phase 3 |

Exceptions: 44px button height (touch target minimum, already established in Phase 2 controls)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 600 | 1.5 |
| Score label | 20px | 600 | 1.2 |
| Result banner | 24px | 600 | 1.2 |

## Phase 3 Interaction Contract

### Split: Active Hand Highlighting

| Property | Value | Source |
|----------|-------|--------|
| CSS class | `bj-hand-area--active` | RESEARCH.md Pattern 6 |
| Effect | `box-shadow: 0 0 12px 4px rgba(230, 184, 0, 0.6)` | RESEARCH.md Pattern 6 |
| Border radius | 8px | RESEARCH.md Pattern 6 |
| Applies to | `.bj-hand-area` container of the active hand index | CONTEXT.md |
| Toggle behavior | Remove from previous hand, add to current hand when `#activeHandIndex` changes | CONTEXT.md |
| When visible | Only during split play (two hands active) | Implicit -- single hand needs no indicator |

### Split: Auto-Advance Between Hands

| Property | Value | Source |
|----------|-------|--------|
| Pause duration | 800ms (`ANIM.HAND_ADVANCE_DELAY`) | RESEARCH.md constants, Claude's discretion |
| Pattern | Identical to round auto-advance -- brief pause, then focus shifts | CONTEXT.md |
| Trigger | Hand 1 completes (bust, stand, or double) | CONTEXT.md |
| Visual | Active hand glow moves from hand 0 to hand 1 | CONTEXT.md |

### Split: Sequential Result Banners

| Property | Value | Source |
|----------|-------|--------|
| Banner duration per step | 1500ms (`ANIM.BANNER_SEQUENTIAL`) | RESEARCH.md constants, Claude's discretion |
| Sequence | Hand 1 result -> Hand 2 result -> Net result | CONTEXT.md |
| Banner styling | Same `.bj-result-banner` class with existing win/loss/push color variants | Phase 2 CSS |
| Net result text | "Net: +$X" / "Net: -$X" / "Net: Push" | Claude's discretion |
| Fade behavior | Same `--bj-result-fade` (300ms) transition between each step | Phase 2 CSS |

### Insurance: Button Swap

| Property | Value | Source |
|----------|-------|--------|
| Trigger | Dealer upcard is ace AND player does not have blackjack | CONTEXT.md |
| Button 1 class | `.bj-btn-insurance` | RESEARCH.md Pattern 5 |
| Button 1 label | "Take Insurance ($X)" where X = half original bet in dollars | Claude's discretion |
| Button 2 class | `.bj-btn-no-insurance` | RESEARCH.md Pattern 5 |
| Button 2 label | "No Insurance" | Claude's discretion |
| Button background | `#d4940a` | CONTEXT.md (amber/gold) |
| Button hover | `#e6a810` | RESEARCH.md CSS example |
| Button text color | `#0a0e1a` | Matches Deal button pattern |
| Button size | `min-width: 120px; height: 44px` | RESEARCH.md Pattern 5 |
| Button font | 14px / weight 600 | Matches existing button pattern |
| Placement | Replaces chip-row and action-row in controls zone | CONTEXT.md |
| Restore | After insurance decision, restore normal controls | Implicit |

### Insurance: Status Message

| Property | Value | Source |
|----------|-------|--------|
| Insurance lost copy | "Insurance lost -$X" | CONTEXT.md |
| Insurance won copy | "Insurance pays +$X" | Claude's discretion (symmetric with lost) |
| Display method | Brief inline status message in result banner position | CONTEXT.md |
| Display duration | 1500ms | Claude's discretion (matches banner sequential timing) |
| Then | Play continues normally (if no dealer blackjack) or round ends (if dealer blackjack) | CONTEXT.md |

### Insurance: Timing Flow

| Step | What Happens |
|------|-------------|
| 1 | Deal completes, dealer upcard is ace, player has no blackjack |
| 2 | Controls zone swaps to insurance buttons |
| 3 | Player clicks Take Insurance or No Insurance |
| 4 | Hole card flips immediately (300ms flip animation) |
| 5a | If dealer blackjack + insurance taken: "Insurance pays +$X" banner, then round over |
| 5b | If dealer blackjack + insurance declined: round over (dealer blackjack wins) |
| 5c | If no dealer blackjack + insurance taken: "Insurance lost -$X" status, then player turn |
| 5d | If no dealer blackjack + insurance declined: player turn continues normally |

### Split Button

| Property | Value | Source |
|----------|-------|--------|
| CSS class | `.bj-btn-split` | Follows existing `bj-btn-*` pattern |
| Label | "Split" | Standard |
| Background | `#2a3a5c` (same as Hit/Stand/Double) | Matches existing action buttons |
| Hover | `#3a4f7a` | Matches existing action buttons |
| Placement | In `.bj-action-row` alongside Hit, Stand, Double | Natural extension |
| Enabled when | `getAvailableActions()` includes 'split' | Engine-driven |
| Disabled styling | `#1a2236` background, `#8899aa` text, `cursor: not-allowed` | Matches existing disabled pattern |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Split button label | "Split" |
| Insurance CTA | "Take Insurance ($X)" |
| Decline insurance | "No Insurance" |
| Insurance lost status | "Insurance lost -$X" |
| Insurance won status | "Insurance pays +$X" |
| Hand 1 result banner | "[Win/Loss/Push] Hand 1: [+$X/-$X/Push]" |
| Hand 2 result banner | "[Win/Loss/Push] Hand 2: [+$X/-$X/Push]" |
| Net result banner (split) | "Net: +$X" / "Net: -$X" / "Net: Push" |
| Single-hand result banner | No change from Phase 2 (e.g. "You win +$150!", "Bust!", "Push") |
| Empty state heading | Not applicable (game always has a table visible) |
| Empty state body | Not applicable |
| Error state | Not applicable (no error states in Phase 3 -- all invalid actions are prevented by disabling buttons) |
| Destructive confirmation | Not applicable (no destructive actions in Phase 3) |

**Note:** All dollar amounts display as whole dollars (integer cents / 100). The "$X" placeholder means the computed dollar value (e.g., "$25", "$150").

| Constant | Value | CSS Property | Usage |
|----------|-------|-------------|-------|
| `ANIM.HAND_ADVANCE_DELAY` | 800ms | `--bj-hand-advance-delay` | Pause between hand 1 completion and hand 2 start |
